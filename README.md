### Задание 2

1. Пользователь вводит данные для пополнения<BR>
Выбирает оператора, вводит номер телефона и сумму. Подтверждает оплату.

2. Система отправляет запрос в платёжный сервис<BR>
Приложение, обрабатывающее пополнение, формирует специальный запрос и отправляет его в банк или платёжную систему.

3. Платёжный сервис проверяет данные<BR>
Проверка  реквизитов, есть ли деньги на счёте, лигитимность операции.

4. Списание денежных средств<BR>
Если всё ок, банк списывает нужную сумму со счёта пользователя. Если денег недостаточно или операция заблокирована, транзакция не проходит.

5. Платёжный сервис отправляет деньги оператору связи<BR>
После успешного списания сервис передаёт информацию и деньги мобильному оператору.

6. Оператор пополняет баланс и высылает подтверждаение операции<BR>
Оператор увеличивает баланс телефона на нужную сумму и посылает обратно подтверждение — смс или ПУШ.

### Задание 3

1. Строгая структура данных (таблицы, типы, связи)<BR>
SQL-базы используют чёткую схему, поэтому данные более организованы и предсказуемы.

2. Мощный язык запросов (SQL)<BR>
SQL — понятный и универсальный язык, который позволяет легко делать сложные выборки, сортировки, объединения таблиц и аналитику.

3. Гарантии целостности данных (ACID)<BR>
SQL-системы обеспечивают надёжность транзакций: данные не потеряются и не "сломаются" даже при сбоях.

4. Поддержка связей между данными<BR>
SQL отлично подходит для задач, где есть много связей — например, пользователи, заказы, товары. Это реализуется через первичные и внешние ключи.

5. Gодходит для сложной аналитики<BR>
Для отчётов, аналитики и сложных запросов SQL-базы обычно работают лучше и эффективнее, чем большинство NoSQL-систем.
